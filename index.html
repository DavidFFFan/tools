峰值明细

载入数据后可查看全量申请、峰值时刻仍占用的内存块，以及点击/框选区域明细。
第 0 / 0 页
单击曲线附近的点会显示该时间点的明细。按住 Alt 拖拽框选后，会同时放大并显示该时间范围的明细。明细表使用自己的横向滚动条，不会把整个网页撑宽。
明细来源说明
  • 从头开始：显示所有有效申请记录，按 start_time_stamp 从小到大排列，用于从任务开始顺序排查内存申请。
  • 无碎片峰值时刻存活：先用 size 按申请/释放时间累加出“无碎片显存”曲线，找到该曲线峰值时间点，再显示这个时间点仍存活的记录。
  • 含碎片峰值时刻存活：先读取 actual_peak_memory 形成“含碎片显存”曲线，找到该曲线峰值时间点，再显示这个时间点仍存活的记录。
  • 点击时刻存活：点击图上的某个时间点后，显示该时间点仍存活的记录；判定条件为 start_time_stamp <= 点击时间 <= end_time_stamp。
  • 点击后续申请：点击图上的某个时间点后，显示 start_time_stamp >= 点击时间 的后续申请记录，适合从某个阶段开始向后排查。
  • 框选时间范围：按住 Alt 框选一段时间后，显示 start_time_stamp 落在该时间范围内的申请记录。
  • 异常 PyNative：在无碎片峰值时刻存活的记录里，筛出 user_tasks 已早于峰值时间结束、但 PyNative 相关内存仍存活的记录，用于辅助排查疑似未及时释放的内存。
